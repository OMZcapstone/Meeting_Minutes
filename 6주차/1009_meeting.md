# 🚗 중간보고 및 구현설계 발표 준비

## 1. 개발 진행 상황

### 👨‍💻 수빈 담당 - 주차라인 인식 시스템

#### 1.1 현재 진행 상황
- **YOLO 라이브러리 활용**: 타이어 인식 시스템 구현
- **라인 인식 알고리즘**: 타이어 근처 흰색 라인 감지
- **문제점**: 적절한 학습모델 부족으로 정확도 저하

#### 1.2 해결 방안
- **AI허브 데이터셋**: 차량 인식용 데이터셋 지속 테스트
- **데이터 수집**: 주차장 직접 촬영을 통한 데이터 보강
- **임시 해결책**: 오늘까지 모델 문제 해결 안 되면 주차장에서 직접 타이어 촬영

#### 1.3 알고리즘 개선안 (근호 제안)
- **단순화된 접근**: 타이어 개별 인식 대신 차량-라인 겹침 분석
- **객체 모델**: 주차라인(흰선), 차량(타이어 포함)으로 단순화
- **판단 로직**: 흰선 위에 차량 감지 시 불법주차로 판단

### 👩‍💻 보은 담당 - 객체인식 구현설계

#### 2.1 진행 상황
- **YOLO 기반 객체인식**: 사람, 주차된 차량, 주행 중인 차량 구현설계
- **노션 페이지**: 설계 내용 정리 완료
- **역할 분담 확인**: 수빈의 구현과 연계하여 설계 진행

#### 2.2 기능 구분
- **2번째 기능** (보은): 장애물 인식을 위한 객체인식 기반
- **3번째 기능** (수빈): 주차라인 인식 및 불법주차 판단

### 🎯 창혁 담당 - 카메라 시스템 설계

#### 3.1 하드웨어 구성
- **카메라 6개**: 전방위 인식 시스템
- **실시간 처리**: 6개 카메라 동시 처리의 기술적 한계 확인

#### 3.2 하드웨어 분산 처리 방안
- **라즈베리파이 2개**: 처리 부하 분산
- **웹캠 4개 + 라즈베리파이 카메라 2개**: 하이브리드 구성
- **노트북 + 라즈베리파이**: 연산 분담
  - 라즈베리파이: 라인 탐지 알고리즘
  - 노트북: 자율주행 알고리즘

#### 3.3 확장성 고려사항
- **HAT 필요**: 라즈베리파이에 2개 이상 카메라 연결 시
- **통신 방식**: 라즈베리파이-노트북 간 데이터 전송 최적화

## 2. 발표 준비 계획

### 📋 발표 구성 방향

#### 2.1 학기별 목표 설정
- **1학기 목표**: 안전 운전 + 센서 파싱
- **2학기 목표**: 소형화 + 자율주행 구현

#### 2.2 기능설계 발표 내용
1. **센서 파싱**: 기본 하드웨어 동작 확인
2. **객체인식**: 장애물 및 안전운전을 위한 인식 시스템
3. **주차라인 인식**: 불법주차 판단 핵심 기능

#### 2.3 구현설계 세부 내용
- **2학기 계획**: 미래 자율주행 기능 포함
- **현재 구현**: 1학기 핵심 기능 중심
- **실연 준비**: 주차라인 인식 알고리즘 시연 영상

### 📊 발표 자료 준비

#### PPT 제작 계획
- **일정**: 일요일 10시 30분 PPT 제작 회의
- **준비물**: 수빈의 라인 인식 알고리즘 실행 영상
- **목표**: 회의 때 실시간 시연 가능한 수준

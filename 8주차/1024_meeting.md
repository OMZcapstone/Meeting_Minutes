# 8주차 회의록 - 멘토링 세션 (2024년 10월 24일)

## 📋 회의 정보

- **날짜**: 2024년 10월 24일
- **시간**: 11시
- **참석자**: 캡스톤 팀원
- **회의 유형**: 멘토링 세션

## 🎯 주요 안건

### 1. 라이다(LiDAR) 시스템 설명

- **현장 시연**: 4채널 라이다를 이용한 거리 측정 데모
  - 물체까지의 거리 측정 가능 (약 6m 거리의 벽면 측정)
  - 포인트 클라우드 생성으로 주변 환경 인식

### 2. 자율주행 로봇 개발 방향

- **로봇 유형**: AMR(Autonomous Mobile Robot) 개발 목표
  - AGV와 달리 동적 판단 및 자율 주행 가능
  - SLAM 기반 지도 작성 및 장애물 회피 기능 구현

### 3. 하드웨어 구성 방안

- **구동부**: 이륜구동 방식 채택
  - 엔코더 모터 사용으로 피드백 제어
  - 터틀봇 참조 모델 검토 권장
  - 10인치 바퀴 및 DC 기어 모터 구매 완료

### 4. 센서 시스템 구성

- **라이다**: RP 라이다 사용 예정 (2-3만원 대)
  - 앞뒤 2개 설치로 전방향 감지
  - 10m 내외 거리 측정 가능
- **카메라**: 6방향 서라운드 뷰 시스템
  - 4개 모듈 + 2개 추가 모듈
- **초음파 센서**: 근거리 물체 감지용 (약 6개)
- **IMU 센서**: 로컬라이제이션 및 자세 제어

### 5. 소프트웨어 개발 환경

- **ROS(Robot Operating System)** 기반 개발 권장
  - 리눅스 환경에서 개발
  - SLAM 패키지 활용으로 개발 시간 단축
  - 모듈화된 개발로 효율성 증대

### 6. 하드웨어 플랫폼 선택

- **메인 컨트롤러**: 라즈베리파이 vs 엔비디아 젯슨 나노
  - 라즈베리파이 5 추천 (메모리 용량 고려)
  - 젯슨 나노도 검토 대상
- **개발 환경**: 초기에는 노트북 활용 권장

### 7. 지하주차장 특화 기능

- **환경 특성**: 고정된 구조물과 가변적인 차량 배치
- **SLAM vs 마커 기반**:
  - 마커 기반 로컬라이제이션 검토
  - 기둥에 QR 코드 부착으로 위치 인식
  - 슬램과 마커 방식 병행 고려

### 8. 개발 전략 및 일정

- **1단계**: 구동부 안정화 (하드웨어 멘토링 집중)
- **2단계**: 센서 통합 및 데이터 처리
- **3단계**: 자율주행 알고리즘 구현
- **분리 개발**: 카트를 이용한 센서 테스트 병행
